## Гипотезы

### HYP-001: Per-intent threshold calibration

- **Задача:** OOS Detection
- **Гипотеза:** Индивидуальная калибровка порога для каждого интента
  (на основе плотности кластера в embedding-пространстве) улучшит
  качество OOS-детекции по сравнению с глобальным порогом.
- **Мотивация:** Разные интенты имеют разную "компактность" в
  embedding-пространстве. Глобальный порог может быть слишком
  строгим для одних интентов и слишком мягким для других.
- **Реализация:** не реализована
- **Метрика:** OOS Recall и In-Domain Accuracy на CLINC150
- **Результат:** —
- **Вывод:** Отложена. AutoML-оптимизация AutoIntent уже обеспечивает
  стабильный глобальный threshold (CV ~3%). Per-intent калибровка
  может дать прирост в сценариях с сильно неравномерной плотностью
  кластеров, но в few-shot режиме (10–20 примеров на интент)
  статистики на кластер недостаточно для надёжной калибровки.
- **Дата:** —

### HYP-002: Asymmetric cost function

- **Задача:** OOS Detection
- **Гипотеза:** Оптимизация порога под асимметричную функцию стоимости
  α·FNR + β·FPR (α > β) улучшит OOS Recall при приемлемой потере
  In-Domain Accuracy, так как для guardrail FN дороже FP.
- **Мотивация:** В стандартном AutoIntent threshold оптимизируется под F1,
  который симметрично взвешивает FN и FP. В guardrail-контексте
  пропущенный OOS-запрос (FN) ведёт к неверному ответу агента,
  тогда как ложно заблокированный in-scope запрос (FP) даёт
  безопасный fallback.
- **Реализация:** notebooks/05_hypothesis_asymmetric_cost.ipynb,
  постфактум подбор threshold на val-сете
- **Метрика:** OOS Recall, In-Domain Accuracy, F1 OOS, FPR
- **Результат:** 20-shot, 3 seeds (mean ± std):
  - 1:1 (baseline): Recall=0.899±0.007, Acc=0.934±0.007,
    F1=0.818±0.007, FPR=0.066±0.007
  - 2:1: Recall=0.937±0.030, Acc=0.883±0.045,
    F1=0.764±0.047, FPR=0.117±0.045
  - 5:1: Recall=0.969±0.010, Acc=0.806±0.052,
    F1=0.685±0.051, FPR=0.194±0.052
  - 10:1: Recall=0.976±0.011, Acc=0.753±0.086,
    F1=0.639±0.071, FPR=0.247±0.086
- **Вывод:** Опровергнута в части "без критической потери In-Domain Acc".
  Механизм работает, но trade-off невыгоден: F1-оптимизированный
  baseline (1:1) уже достигает Recall=0.899 при FPR=0.066.
  Переход к 2:1 нестабилен (std Acc=0.045). При α≥5 каждый
  пятый легитимный запрос блокируется. Асимметричная cost function —
  configurable knob для специфических требований, но не улучшение
  пайплайна по умолчанию.
- **Дата:** апрель 2026
