- **Вывод:** Переход на knn — **необходимое, но НЕ достаточное** условие

## Рекомендации

Поскольку проблема в scoring-узле, а не в пороге:

1. **Увеличить n_shots** (100+) — текущие 10-50 недостаточны
2. **Улучшить embedder** — domain-specific fine-tuning
3. **Сбалансированный few-shot sampling** — не 10:1
4. **Two-stage подход** — coarse filter + fine classifier
5. **Калибровка** (Platt/isotonic) — для интерпретируемых скоров
6. **Зафиксировать knn** — linear схлопывает скоры
